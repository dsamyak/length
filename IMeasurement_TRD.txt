TECHNICAL OVERVIEW1.1 Architecture SummaryIMeasurement is a single-page React application (SPA) built with Vite as the bundler. It follows the architecture established in the dsamyak/numberbound GitHub repository, adapted for the Length topic. The application is fully client-side with no backend database required — all question data, state management, and scoring occurs in-browser.The app is structured into 6 distinct 'phase' components, each rendered conditionally based on a global phase state. Audio management is centralised via a custom useAudio hook wrapping Howler.js. Animations are powered by Framer Motion for page transitions and component-level effects.1.2 Core Technology StackLayerTechnologyVersion / NotesFrameworkReact18.xBundlerVite5.xStylingTailwind CSS3.x + custom CSS variablesAnimationFramer Motion11.xAudioHowler.js2.xState ManagementReact Context + useReducerBuilt-in (no Redux needed)Drag & Dropreact-beautiful-dnd or @dnd-kit/coreLatest stableIconsLucide React + custom SVGsLatest stableRoutingReact Router v6For phase navigation (optional: useState phase manager)DeploymentVercelvercel.json config includedCode StyleESLint + PrettierAirbnb config2. i > 0; i--) {    const j = Math.floor(Math.random() * (i + 1));    [arr[i], arr[j]] = [arr[j], arr[i]];  }  return arr;}export function selectQuestions(pool, count = 20) {  return shuffle(pool).slice(0, count);}// For each multiple-choice question, answer options are also shuffled:export function shuffleOptions(question) {  return { ...question, options: shuffle(question.options) };}3.4 useAudio HookWraps Howler.js for centralised audio management:// hooks/useAudio.jsimport { Howl } from 'howler';const sounds = {  correct: new Howl({ src: ['/audio/sfx/correct.mp3'] }),  wrong:   new Howl({ src: ['/audio/sfx/wrong.mp3'] }),  bgMusic: new Howl({ src: ['/audio/music/bg_loop.mp3'], loop: true, volume: 0.3 }),  celebrate: new Howl({ src: ['/audio/sfx/celebrate.mp3'] }),};export function useAudio(audioEnabled) {  const play = (key) => { if (audioEnabled) sounds[key]?.play(); };  const playNarration = (src) => {    if (audioEnabled) new Howl({ src: [src], autoplay: true });  };  return { play, playNarration };}4. SIMULATION COMPONENTS — TECHNICAL SPEC4.1 RulerDrag.jsxDrag-to-compare simulation using mouse/touch drag events:Two coloured rectangular bars (CSS div elements with dynamic width via useState)Drag handles on right side of each bar — pointer events update bar width on dragOn 'Compare' button click: bars snap to same left baseline, longer bar gets gold glowState: { barAWidth, barBWidth, compared, result }Framer Motion: bars animate to baseline position on compareTouch support: onTouchMove events alongside onMouseMove for tablet use4.2 SortQueue.jsxDrag-to-sort simulation using @dnd-kit/core:DndContext wraps a SortableContext with 3–4 object componentsEach object is a useSortable() item rendered as an image cardOn drop: array reorders, Framer Motion animates card positionsValidation: correct sorted order triggers celebratory Framer Motion sequenceIncorrect: cards gently shake, 'Try again!' message appearsState: { items[], sorted, validated }4.3 GuessLength.jsxBinary choice simulation (Longer / Shorter tap):Reference object rendered on left; mystery object rendered on rightTwo large buttons: 'LONGER 📏' and 'SHORTER ✂️'On click: correct answer triggers green glow + audio; wrong triggers gentle shake + audioAfter 2s feedback delay, next round loads automatically3 rounds total per simulation visitState: { currentRound, score, answered, isCorrect }5.
